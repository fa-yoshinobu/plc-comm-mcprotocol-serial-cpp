# RJ71C24-R2 RS-232C Report

Audience: maintainers and users who need the current validated view of the `RJ71C24-R2` path.

This page is the single report for this hardware target. Older dated `Format4` and `Format5` logs
were consolidated here so the current state does not fragment across multiple files.

## Target

- PLC CPU: Mitsubishi iQ-R `R08CPU`
- Serial module: `RJ71C24-R2`
- Link: `RS-232C`

## Current Validated Setups

### Format4 ASCII

- Settings: `19200 / 8E1 / MC Protocol Format4 ASCII / CRLF / sum-check off / station 0`
- Validated scope:
  - `cpu-model`
  - `loopback`
  - contiguous `0401/1401` read/write
  - host buffer `0613/1613`
  - module buffer `0601/1601`
  - qualified helper `read-qualified-words` / `write-qualified-words`
- Operational note:
  - after timeout or mixed-response states, ASCII `EOT CRLF` or `CL CRLF` recovers the C24
    transmission sequence

### Format5 Binary

- Settings: `28800 / 8E2 / MC Protocol Format5 Binary / sum-check on / station 0`
- Practical CLI family selection: `--series ql`
- Validated scope:
  - contiguous `0401/1401` read/write
  - native `0403`
  - native `1402` words
  - native `1402` bits
  - native `0406`
  - native `1406`
  - native `0801/0802`

## Series Selection Note

On `RJ71C24-R2 + R08CPU`, the practical native subcommand family for random, write, multi-block,
and monitor traffic is `--series ql`.

Observed follow-up behavior:

- `--series ql`: native `0403`, `1402`, `0406`, `1406`, `0801`, and raw `0802` pass
- `--series iqr`: false negatives remain on random and monitor traffic

Keep `--series iqr` only as a probe mode on this target.

## Qualified Access Note

Qualified helper access remains the practical public path on this target:

- helper `read-qualified-words 'U3E0\G10' 1` returned `0x83BD`
- helper `read-qualified-words 'U3E0\HG20' 1` returned `0x0000`
- helper `write-qualified-words 'U3E0\HG20' ...` readback and restore passed

Native qualified access is still unresolved:

- historical probes produced `0x7F22`, timeout, `0x4031`, and semantically mismatched success
- no native qualified read/write path should be treated as validated
